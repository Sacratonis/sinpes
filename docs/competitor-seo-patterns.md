# SINPES Competitor SEO Pattern Library

Observed targets:

- dafontfree.io
- dafont.com
- 1001fonts.com

This document records reusable public patterns. It must not contain copied
descriptions, fabricated demand, or unsupported claims about a font.

Implementation status and the complete working checklist live in
`docs/enterprise-seo-strategy.md`.

## What each competitor does best

### DafontFree

- Publishes frequently and exposes dates prominently.
- Uses keyword-rich font titles and category URLs.
- Uses short archive excerpts of roughly 25–35 words.
- A sampled font detail description contained 173 words.
- The sampled detail page used Article, WebPage, ImageObject,
  BreadcrumbList, WebSite, Organization, and Person schema entities.
- Descriptions commonly follow: category, visual positioning, use cases,
  included styles, and a download action.

### DaFont

- Wins through inventory, age, direct downloads, and extremely fast browsing.
- Uses a two-level theme taxonomy.
- Shows author, category, variants, glyph count, first-seen date, download
  count, recent-download count, and usage status close to the download action.
- Font-page editorial copy is often minimal; author notes and utility carry
  the page.
- Offers filters for variants, character support, popularity, and recency.

### 1001 Fonts

- Uses the deepest observed taxonomy.
- Organizes discovery by category, size, weight, width, occasion, holiday,
  decade, historical style, modern style, attitude, handwriting, theme,
  symbols, and popularity.
- A sampled font description contained 201 words.
- Font pages expose each variant, multiple specimen sizes, character maps,
  language coverage, technical statistics, related tags, author pages,
  downloads, favorites, and related fonts.
- Search and filter settings can be preserved in a permalink.

## SINPES description pattern

Target 160–220 words when verified metadata supports that depth.

1. Identify the family and verified category.
2. State the number of files, styles, or weights using registry data.
3. Explain practical use cases without inventing anatomy or visual traits.
4. Mention supported formats and real capabilities.
5. Give a short recommendation for applying the family.
6. Link to its category, related collections, and relevant articles.

Archive cards should use a separate 25–35 word excerpt rather than truncating
the full description randomly.

## Keyword taxonomy to test

Every landing page still requires real inventory and demand evidence.

- Classification: serif, sans serif, slab serif, script, display, monospace.
- Size: poster, headline, body text, caption, small text.
- Weight: thin, light, regular, medium, bold, black.
- Width: condensed, narrow, regular, wide, extended.
- Occasion: wedding, birthday, party, tattoo.
- Holiday: Christmas, Halloween, Valentine, Easter.
- Era: 1920s through 1990s, retro, vintage, Art Deco, Art Nouveau.
- Modern: geometric, futuristic, digital, pixel, techno, stencil.
- Style: calligraphy, comic, elegant, grunge, outlined, distorted.
- Attitude: quirky, romantic, groovy, playful, street, luxury.
- Use case: UI, web, editorial, branding, posters, packaging, magazines.
- Technical: variable, high character coverage, multilingual, large families.
- Discovery: new fonts, trending fonts, popular fonts, largest families.

## Rules for programmatic pages

- Require at least four accurately matching fonts.
- Require a distinct primary intent.
- Block exact keyword and intent cannibalization.
- Use unique introductions and selection guidance.
- Show real specimens and useful filters.
- Link every page into a browsable hierarchy.
- Never create empty, near-duplicate, or keyword-swapped doorway pages.

## Bing implementation

- Keep Bing Webmaster verification on every public page.
- Notify IndexNow only after the related deployment succeeds.
- Submit new, updated, unpublished, or deleted font and article URLs.
- Keep the IndexNow verification key at the site root.
- Retain failed URL notifications for the next retry.
- Continue importing real Bing query impressions into the Oracle.

## Monitoring cadence

- Weekly: taxonomy changes, new category URLs, title patterns, archive excerpt
  lengths, description lengths, and navigation changes.
- Monthly: competitor inventory estimates, new landing-page types, prominent
  filters, internal-link modules, and newly targeted search intents.
- Quarterly: decide which verified patterns improved SINPES impressions,
  indexing, rankings, clicks, and downloads.
